1. Active featured/variant cards still reload the already-active model on every tap. The UI labels an active download as `Model Ready`, but the button stays enabled in both featured and variant cards, and `onVariantAction()` routes any `DOWNLOADED` state straight to `useModel()`; `useModel()` then force-reloads the model with `loadModel(forceReload = true)`. On the recommendation screen this turns a harmless tap on Most Recommended, Fastest, or Higher Quality into an unnecessary full reload of the active model.

2. The recommendation screen still lacks meaningful accessibility semantics for its section structure and card-like actions. `Header()` and `SectionTopBar()` render section titles like Most Recommended, Fastest Option, and Higher Quality Option as plain text with no heading semantics, so assistive tech cannot navigate the screen by section. `ActionCard()` is exposed as a generic clickable container rather than a button role, so controls like Refine Preferences and Browse Other Models are not announced as actionable controls.
